## 二、网络配置

### 2.1 USB 连接拓扑

| 设备 | IP 地址 | 说明 |
|------|---------|------|
| 本地 PC（USB 口分配） | `192.168.55.100` | USB 连接后自动分配 |
| NX 应用板 | `192.168.55.1` | 用户 `mi`，密码 `123` |
| 运控板（MCU） | `192.168.55.233` | 通过 Type-C 线和 CyberDog 的 Download 接口连接 |

### 2.2 USB 多播配置

```bash
# 确认 USB 设备名称（通常为 usb0）
ifconfig | grep -B 1 192.168.55.100 | grep "flags" | cut -d ':' -f1

# 配置多播（必须，否则 LCM 通信失败）
sudo ifconfig usb0 multicast

# 添加路由表（224.0.0.0 为 LCM 多播地址）
sudo route add -net 224.0.0.0 netmask 240.0.0.0 dev usb0
```

### 2.3 运控板连接验证

```bash
ping 192.168.55.100     # 本地 PC 的 USB IP
ssh mi@192.168.55.1     # 登录 NX 应用板
ping 192.168.55.233     # 运控板 MCU（通过 USB 连接后）
```

## 三、电机 SDK 编译与部署

### 3.1 NX 应用板编译

```bash
# 在 NX 应用板上操作
ssh mi@192.168.55.1

cd ~/cyberdog_motor_sdk
mkdir build && cd build
cmake ..
make -j4
./Example_MotorCtrl   # 测试（前台运行）
# nohup ./Example_MotorCtrl &   # 后台运行
```

### 3.2 部署到运控板（MCU）

```bash
# 在 NX 上打包
scp -r cyberdog_motor_sdk mi@192.168.55.1:/home/mi/

# 登录运控板
ssh root@192.168.55.233
mkdir /mnt/UDISK/cyberdog_motor_sdk

# 从 NX 复制到运控板
#（需要在 NX 上执行）
scp libcyber_dog_motor_sdk.so Example_MotorCtrl root@192.168.55.233:/mnt/UDISK/cyberdog_motor_sdk

# 在运控板上运行
ssh root@192.168.55.233
cd /mnt/UDISK/cyberdog_motor_sdk
export LD_LIBRARY_PATH=/mnt/UDISK/cyberdog_motor_sdk
./Example_MotorCtrl
```

### 3.3 ⚠️ LCM 通信失败

```
Log prompt: Motor control mode has not been activated successfully.
```

**原因**: LCM 通信未建立，电机控制模式无法激活。
**解决**: 确认 USB 多播配置 + 运控板网络正常。

## 四、运控板进程管理

### 4.1 关闭用户程序 → 关闭主程序顺序

> ⚠️ 重要安全提醒：关闭程序时必须先关闭用户程序，触发主程序（ctrl）超时保护趴下，再关闭主程序。同时关闭主程序和用户程序，电机会因 CAN 总线超时位置锁定，再次启动易发生危险。

```bash
# 1. 先关闭用户程序（Example_MotorCtrl）
ssh root@192.168.55.233
ps | grep Example_MotorCtrl | grep -v grep | awk '{print $1}' | xargs kill -9

# 2. 等待主程序超时保护触发（趴下）

# 3. 再关闭主进程
ps | grep -E 'manager|ctrl|imu_online' | grep -v grep | awk '{print $1}' | xargs kill -9
```

### 4.2 重启运控程序

```bash
ssh root@192.168.55.233
export LD_LIBRARY_PATH=/mnt/UDISK/robot-software/build
nohup /mnt/UDISK/manager /mnt/UDISK/ >> /mnt/UDISK/manager_log/manager.log 2>&1 &
```

### 4.3 重启运控板系统

```bash
ssh root@192.168.55.233 "reboot"
```

## 五、开机自启动配置

通过配置文件添加用户程序到开机自启动列表：

编辑 `/mnt/UDISK/manager_config/fork_para_conf_lists.json`，在进程管理文件中添加（注意结尾逗号）：

```json
"600003": {
    "fork_config": {
        "name": "Example_MotorCtrl",
        "object_path": "/cyberdog_motor_sdk/",
        "log_path": "",
        "paraValues": ["", "", ""]
    }
}
```

添加后重启运控程序生效。

## 六、ROS2 编译

### 6.1 基础编译

```bash
mkdir -p ros_apps/src
cd ros_apps/src
git clone https://github.com/MiRoboticsLab/cyberdog_ros2.git
cd ..

# 编译全部包
colcon build --merge-install

# 或编译到指定目录
export OUTPUT_DIR=/opt/ros2/cyberdog
colcon build --merge-install --install-base $OUTPUT_DIR
```

### 6.2 仅编译最小功能包（影响开机和运动）

```bash
colcon build --merge-install --packages-up-to cyberdog_bringup
```

### 6.3 编译选项

```bash
# 小爱同学支持（默认关闭）
colcon build --cmake-args -DXIAOMI_XIAOAI=ON

# 国内加速
colcon build --cmake-args -DBUILD_INSIDE_GFW=ON
```

### 6.4 部署

```bash
# 如果使用 /opt/ros2/cyberdog 路径，且在铁蛋本机编译
sudo systemctl restart cyberdog_ros2.service
```

## 九、注意事项

1. **关闭程序顺序**：必须先用户程序 → 再主程序，否则 CAN 总线锁死
2. **USB 多播**：不配置则 LCM 通信失败，电机无法激活
3. **运控板 vs 应用板**：
   - NX 应用板 = `ssh mi@192.168.55.1`，运行 ROS2
   - 运控板 MCU = `ssh root@192.168.55.233`，运行电机控制固件
4. **Mode 1 开发者模式**：需要在运控板配置文件 `/mnt/UDISK/robot-software/config/user_code_ctrl_mode.txt` 设置为 `1` 才能解锁所有动作
